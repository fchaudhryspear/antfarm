## Root Causes

### RC-1: Poisoned `task` Variable (Primary Cause — 7/10 agents affected)

**What happened:** Run #92 was dispatched with:
```
task = "Bug #14 VALIDATION — Flobase Capital Platform full review. Verify consolidate receives all 10 agent outputs via namespaced storage."
```

Every step input template starts with `TASK: {{ task }}`, so all 10 agents received "Bug #14 VALIDATION" and "Verify consolidate receives all 10 agent outputs" as their primary instruction. Three agents (Backend, Docs, Performance) latched onto this and produced meta-commentary about the pipeline instead of reviewing code. Security, UX, and Code-Quality couldn't find the repo because they lacked filesystem navigation instructions.

**Fix (v3.2):** Step inputs now start with a positive scope preamble:
```
You are reviewing the codebase at {{ repo_path }}.
Scope: {{ scope }}
Focus: {{ focus_areas }}
```
Followed by explicit anti-drift clause and mandatory filesystem navigation command. The `{{ task }}` variable is no longer included in step inputs — it was the contamination vector.

### RC-2: Agents Didn't Navigate to the Codebase (3/10 agents: Security, UX, Code-Quality)

**What happened:** Agents received `REPO_PATH: ~/.openclaw/workspace/flobase-capital-platform` as text but never ran `cd` or `ls` on it. They looked only in their own workspace directory (which contains only AGENTS.md) or reported "missing repo_path."

**Fix (v3.2):** Every agent AGENTS.md now has a `## MANDATORY FIRST STEP` section with explicit shell commands to `cd` into the repo and `find` files. Every step input also includes a `MANDATORY FIRST STEP:` instruction with agent-specific file discovery commands.

### RC-3: `~` Tilde in Repo Path (Ambiguous — may not expand in all contexts)

**What happened:** Workflow v3.1 used `repo_path: "~/.openclaw/workspace/flobase-capital-platform"`. Tilde expansion depends on shell context — in a string passed to an LLM, `~` is just a character, not a path.

**Fix (v3.2):** Changed to absolute path: `repo_path: "/Users/faisalshomemacmini/.openclaw/workspace/flobase-capital-platform"`

### RC-4: Product-Review Used Wrong Output Format

**What happened:** Product-review output 467 bytes using `STATUS: done` / `CHANGES:` / `TESTS:` format instead of the required `SCORE:` / `CATEGORY:` / `FINDINGS:` format. This means the consolidate step couldn't parse its output. Not a truncation bug — the agent followed a different output convention entirely.

**Fix (v3.2):** All agent AGENTS.md files now include:
```
**Do NOT use STATUS:/CHANGES:/TESTS: format. Use SCORE:/CATEGORY:/FINDINGS: only.**
```

### RC-5: Performance-Engineer Sub-Agent Overhead (1/10 agents)

**What happened:** Performance-engineer was configured to spawn 3 sub-agents (bottlenecks, load, optimization), merge their outputs, and produce a summary. This took 60-90+ minutes and timed out 4 times in Run #92 before succeeding on attempt 4 at the 90-minute mark.

**Fix (v3.1, already deployed):** Rewrote to direct single-pass analysis. Max 5 findings, 5-minute budget, no sub-agents.
